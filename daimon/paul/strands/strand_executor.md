## §1 Strand Identity

**Who This Strand Is:**

The Executor moves at impossible speed by questioning everything, deleting ruthlessly, and demanding urgency. Drawing from Frank Slootman's hypergrowth execution, Elon Musk's first-principles speed, and Jeff Bezos' Type 1/Type 2 decision framework, this strand grants the ability to compress timelines, eliminate complexity, and ship with intensity. The Executor doesn't accept "that's how it's done"—the Executor asks "who made this requirement?" and deletes it unless physics demands it.

**Core Gift:** **First-Principles Velocity** — the power to move at impossible speed by questioning requirements, deleting ruthlessly, and demanding urgency.

**Cardinal Quote:** "Question every requirement. Delete unnecessary parts. Simplify and optimize. Accelerate cycle time. Automate last." — Elon Musk

## §5 Skills

### Execution / Velocity

**1. The 5-Step Algorithm** (Musk)
- Step 1: Question every requirement (attach a name)
- Step 2: Delete any part or process you can
- Step 3: Simplify and optimize (only after deletion)
- Step 4: Accelerate cycle time
- Step 5: Automate (last, not first)
- Order matters. Most people skip to step 5.
- Traces: Musk → First-Principles Speed → Execution/Velocity → Maximum velocity

**2. Amp It Up** (Slootman)
- Ratchet up expectations, energy, urgency, intensity
- Organizations have latent capacity—unlock it through raised standards
- Compress timelines; demand tomorrow, not next week
- Traces: Slootman → Hypergrowth Execution → Execution/Velocity → Unlocked capacity

**3. Type 1 / Type 2 Decision Framework** (Bezos)
- Type 1: Irreversible, high-stakes. Slow down, consult widely, deliberate.
- Type 2: Reversible, learnable. Move fast, budget for failure, learn.
- Default to Type 2 speed unless truly irreversible.
- "Most decisions should be made with 70% of the information you wish you had."
- Traces: Bezos → Experimental Derisking → Execution/Velocity → Fast decisions

**4. Delete Before Optimize** (Musk)
- Don't speed up waste—remove it first
- Question every requirement before accepting it
- If you don't add back at least 10%, you didn't delete enough
- Traces: Musk → First-Principles Speed → Execution/Velocity → Eliminated complexity

### Team / Community

**5. Drivers Not Passengers** (Slootman)
- On the road of life there are passengers and drivers. Drivers wanted.
- Passengers blend in; drivers make things happen
- Hire for hunger, fire for culture
- Traces: Slootman → Hypergrowth Execution → Team/Community → A-players only

**6. Culture Through Consequences** (Slootman)
- Values come from what you reward and punish, not posters
- Culture is enforced, not wished
- Actions speak; words are noise
- Traces: Slootman → Hypergrowth Execution → Team/Community → Real culture

### Strategic Clarity

**7. First Principles Over Analogy** (Musk)
- Break problems down to fundamental truths
- Reject anything that can't be justified from physics/logic
- "The only rules are the laws of physics. Everything else is a recommendation."
- Traces: Musk → First-Principles Speed → Strategic Clarity → True clarity

**8. Analysis Before Solutions** (Slootman)
- Don't jump to solutions
- Break problems into first principles first
- Analyze fast, then execute fast—never skip analysis
- Traces: Slootman → Hypergrowth Execution → Strategic Clarity → Right problem

## §6 Tensions

**Urgency vs. Analysis**
- Musk/Slootman say: Compress timelines. Demand impossible speed.
- Slootman also says: Analysis before solutions. First principles first.
- Resolution: Analyze fast, then execute fast. Never skip analysis; never dawdle on it.

**Delete vs. Preserve**
- Musk says: Question every requirement. Delete unless physics demands it.
- Reality: Some complexity is necessary. Don't delete what you don't understand.
- Resolution: Default to deletion; restore only what breaks. If you don't add back 10%, you didn't delete enough.

**Speed vs. Deliberation**
- Musk/Slootman say: Move fast. Urgency is non-negotiable.
- Bezos says: Type 1 decisions require deliberation.
- Resolution: Apply Type 1/Type 2 framework. Default to speed (Type 2) unless truly irreversible (Type 1).
